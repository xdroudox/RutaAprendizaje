# TDD - Test Driven Development

Metodologia donde las pruebas se escriben ANTES del codigo de produccion.

## Ciclo Red-Green-Refactor

1. **RED:** Escribir prueba que falle
2. **GREEN:** Codigo minimo para que pase
3. **REFACTOR:** Mejorar codigo manteniendo pruebas verdes

```
[RED] -> [GREEN] -> [REFACTOR] -> [RED] -> ...
```

## Ejercicios

1. **FizzBuzz con TDD (escribir test primero)**
   **Ejecuta:** `python scripts/runner.py 7 3 1`

2. **Validador de email con TDD**
   **Ejecuta:** `python scripts/runner.py 7 3 2`

3. **Calculadora de promedios con TDD**
   **Ejecuta:** `python scripts/runner.py 7 3 3`
