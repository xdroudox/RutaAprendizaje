Ejercicio 1: Prevenir XSS (Cross-Site Scripting)

SOLUCION: Usar textContent en lugar de innerHTML
function ejercicio1() { var input = document.getElementById('input-xss-sol'); var resultado = document.getElementById('resultado-ej1-sol'); var texto = input.value; // SEGURO: textContent no interpreta HTML resultado.textContent = 'Contenido: ' + texto; }
Explicacion: innerHTML interpreta el string como HTML y ejecuta cualquier script incrustado. textContent trata todo como texto plano, escapando caracteres HTML automaticamente. Si el usuario escribe <script>alert('XSS')</script>: - innerHTML: ejecuta el script - textContent: muestra el texto literal en pantalla

Ejercicio 2: Sanitizar entrada del usuario

SOLUCION: Validar URL antes de usarla
function ejercicio2() { var input = document.getElementById('input-url-sol'); var resultado = document.getElementById('resultado-ej2-sol'); var url = input.value.trim(); var urlLower = url.toLowerCase(); if (urlLower.startsWith('https://') || urlLower.startsWith('http://')) { if (urlLower.includes('javascript:') || urlLower.includes('data:')) { resultado.textContent = 'Error: URL no permitida (contiene protocolo peligroso)'; resultado.style.background = '#f8d7da'; } else { resultado.innerHTML = '<a href="' + url + '" target="_blank">Ir a ' + url + '</a><br>URL valida y segura'; resultado.style.background = '#d4edda'; } } else { resultado.textContent = 'Error: URL debe comenzar con http:// o https://'; resultado.style.background = '#f8d7da'; } }
Explicacion: Se verifica que la URL comience con http:// o https:// (protocolos seguros). Adicionalmente, se bloquean protocolos peligrosos como javascript: (ejecuta codigo) y data: (puede contener payloads). Siempre usa startsWith() para validar el inicio de la URL, no solo includes().

Ejercicio 3: Simular CSRF y su prevencion

SOLUCION: Token CSRF aleatorio + verificacion
var csrfTokenSol = ''; function generarToken() { csrfTokenSol = Math.random().toString(36).substring(2, 15) + Math.random().toString(36).substring(2, 15); document.getElementById('csrf-token-input-sol').value = csrfTokenSol; } function ejercicio3() { var resultado = document.getElementById('resultado-ej3-sol'); var tokenInput = document.getElementById('csrf-token-input-sol').value; var cuenta = document.getElementById('cuenta-destino-sol').value; var monto = document.getElementById('monto-sol').value; if (tokenInput === csrfTokenSol) { resultado.textContent = 'Transferencia de $' + monto + ' a cuenta ' + cuenta + ' exitosa (token CSRF valido)'; resultado.style.background = '#d4edda'; } else { resultado.textContent = 'Error: CSRF detected - token invalido'; resultado.style.background = '#f8d7da'; } }
Explicacion: Se genera un token CSRF aleatorio usando Math.random().toString(36) (base36: numeros + letras). El token se almacena en una variable y se muestra al usuario. Al enviar el formulario, se compara el token enviado con el almacenado. Si no coinciden, significa que la peticion podria venir de otro sitio (CSRF). En produccion, el token se genera en el servidor y se asocia a la sesion del usuario.
